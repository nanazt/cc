# Phase 19.1 Plan 02: Research and Preferences Pipeline Summary

**One-liner:** Structured [DELTA:new/known/varies][HOOK:yes/no] inline tags replacing free-text delta assessment, hook recommendation flow from researcher through user confirmation in preferences, and .state/ persistence for research and preferences outputs.

## What Was Built

### Task 1: agents/convention-researcher.md — Structured Delta Tags and Hook Recommendation

Updated the researcher agent with these specific changes:

- **Input Contract table:** Added `<convention_tools>` row for MCP tool names from cckit.json
- **Step 2: Research Best Practices:** Added instruction to use `<convention_tools>` MCP tools alongside default WebSearch/WebFetch when convention_tools contains tool names
- **Step 4: Replaced** "Identify Delta Test Candidates" (free-text HIGH/LOW/NO-DELTA) with "Classify Practices with Structured Tags" — defines [DELTA:new], [DELTA:known], [DELTA:varies] and [HOOK:yes], [HOOK:no] tags; both tags appear inline on the same line as the practice name heading; classification principles specify that style divergence goes to preferences (not researcher) and hook classification errs toward [HOOK:no] on ambiguity
- **Step 4a: Added** "Recommend Hook Necessity" — after classifying all practices, assess whether a PreToolUse hook is warranted (requires ≥2 [HOOK:yes] practices, clear trigger event, meaningful mechanical validation); produce `## Hook Recommendation` section in output
- **Output Contract Best Practices:** Format updated to show `[DELTA:new|known|varies] [HOOK:yes|no]` inline with practice name; `Delta:` field removed; `Reasoning:` field covers both delta and hook classification
- **Output Contract Hook Recommendation:** New `## Hook Recommendation` section added with Recommended/Trigger event/Hookable rules/Reasoning fields
- **Delta Test Summary table:** Updated to use [DELTA:new], [DELTA:varies], [DELTA:known] rows instead of HIGH/LOW/NO; added [HOOK:yes] and [HOOK:no] rows
- **Return Protocol:** Added `Hook recommended: {yes/no}` to the `## RESEARCH COMPLETE` line
- **Quality Gate:** Removed "Delta assessment provided for every practice with one-sentence reasoning" (superseded); added 5 new tag-specific verification items

### Task 2: skills/convention/step-research.md — .state/ Reads, convention_tools Pass-Through, .state/ Write

- **Step 1:** Added init.json reading at start — reads `convention_tools`, `artifact_type`, `area`, `lang` from `{state_dir}/init.json` via cat command to confirm values before research dispatch
- **Step 2 agent dispatch prompt:** Updated `<research_tools>` description to reference "init.json or step-init pass-through"; added `<convention_tools>` tag in dispatch prompt with value from init.json
- **Step 4 delta display:** Replaced HIGH/LOW/NO-DELTA format with [DELTA:new/known/varies] tag display; added hook summary line showing [HOOK:yes] count and hook recommendation from researcher
- **Step 4a: Added** "Persist Research Results" — uses Write tool to write `research_results` to `{state_dir}/research.md` for downstream step consumption without implicit context dependency
- **Step 5 transition:** Added `state_dir` and `artifact_type` to both "Research first" and "Preferences first" pass-forward lists; added note that research is also persisted at `{state_dir}/research.md`

### Task 2: skills/convention/step-preferences.md — Hook Confirmation and .state/ Write

- **Step 2a: Added** "Hook Confirmation (When Research Recommends Hook)" — skipped when research has no `## Hook Recommendation` or recommendation is "No"; when hook recommended, presents hookable rules list and trigger event in conversation text, then asks via AskUserQuestion "Yes, generate hook + tests" or "No, convention only"; stores `hook_confirmed` boolean and `hook_rules` list
- **Step 3 summary format:** Added conditional `- Hook generation: {Yes — {N} hookable rules / No}` line when `hook_confirmed` is set
- **Step 3a: Added** "Persist Preferences" — uses Write tool to write preferences JSON to `{state_dir}/preferences.json` with `preferences` array, `hook_confirmed`, `hook_rules`, and `artifact_type` fields
- **Step 4 transition:** Added `state_dir`, `artifact_type`, `hook_confirmed`, `hook_rules` to both "Research first" and "Preferences first" pass-forward lists

## Example Diversity Strategy

| Option | Description | Selected |
|--------|-------------|----------|
| Rotating diverse domains + per-site adaptation instruction | Each major example uses a different domain (CLI, web, library) + "adapt to host project" instruction | Initially selected, then superseded |
| Single neutral domain | One universal domain (e.g., file/document ops) for all examples | |
| Abstract placeholders | [entity], [error], [outcome] placeholders only | |

**User's choice:** Initially selected "Rotating diverse domains" but raised concern that rotating still anchors Claude to the specific domains shown. **Notes:** User's key insight: "Even with rotating, Claude is still limited to thinking in terms of whatever domains the examples use." This led to rejecting all concrete-example approaches.

**tools/schema-parser.ts:**

Imports:
```typescript
import { unified } from "npm:unified@^11.0.0";
import remarkParse from "npm:remark-parse@^11.0.0";
import remarkGfm from "npm:remark-gfm@^4.0.0";
import { toString } from "npm:mdast-util-to-string@^4.0.0";
import type { Heading, Root, Table, List } from "npm:@types/mdast@^4.0.0";
```

**Core exported function:**
```typescript
export function parseSchema(source: string): { data?: SchemaOutput; errors: ValidationError[] }
```

When validation succeeds (errors array empty), `data` contains the parsed schema. When validation fails, `data` is undefined and `errors` contains all validation failures. Mutually exclusive outputs (per Research recommendation).

**Parsing pipeline (order matters):**

1. **Parse AST:** `unified().use(remarkParse).use(remarkGfm).parse(source)` -- MUST include remarkGfm for table parsing (Pitfall 1).

2. **Collect H2 sections:** Walk top-level children, find all `heading` nodes with `depth === 2` (Pitfall 4: ignore H1). For each H2, record: heading text (case-insensitive via `toString(node).toLowerCase().trim()`), position, and the slice of children until the next H2.

3. **Parse Meta:** Find section with heading matching `meta` (case-insensitive, Pitfall 3). Find the first `table` child node. Extract rows using defensive cell access (Pitfall 2: `cells[colIndex] ?? ""`). Map key-value pairs:
   - `version` -> parse as integer, default 1
   - `rule-prefix` -> string, default "CR"
   - `e2e-flows` -> parse "true"/"false" as boolean, default false
   - If no Meta section found: use all defaults (parsing rule 4).

4. **Parse Components:** Find section with heading matching `components`. Find first `table` child. Extract header row to identify column indices for "Component", "Description", "Type" (any order). Extract data rows with defensive cell access. Validate:
   - No duplicate component names (error if found)
   - Type names are kebab-case when non-empty (error if invalid, with line number from row position)

5. **Parse Section blocks:** Find all sections with headings matching `/^sections:\s*(.+)$/i`. For each:
   - Extract type name from heading (e.g., "default", "api-gateway")
   - Validate type name is kebab-case (except "default")
   - Find `### Context Sections` sub-heading, extract the numbered list below it. For each list item: extract `strong` node as section name, text after ` -- ` as guide text. Use AST structure for name extraction (Pitfall 6: don't split on ` -- `).
   - Find `### Conditional Sections` sub-heading (optional per D-02). If present, extract bullet list. For each item: extract `strong` node as name, text after `Include when:` as condition.
   - If `### Conditional Sections` is missing: empty conditional array for that type.

6. **Validate cross-references:**
   - Every non-empty Type value in components must have a corresponding section block (error: `Type "{type}" in Components table has no matching "## Sections: {type}" block`)
   - If `## Sections: default` is missing AND components exist, log a warning but use built-in 7+2 fallback (parsing rule 5, Pitfall 7). Intended behavior: the block should be present, but the parser is defensive.
   - Orphaned section blocks (type name in Sections heading but no component references it): warning, not error.

7. **Build output:** Assemble `SchemaOutput` object with `meta`, `components` array (preserving declaration order), and `sections` object keyed by type name.

**CLI entrypoint** (behind `import.meta.main`):
- No args: `console.error("Usage: schema-parser.ts <schema-path>")` then `Deno.exit(1)`
- Read file: `Deno.readTextFile(Deno.args[0])`
- Parse: `parseSchema(source)`
- If errors: `console.error(JSON.stringify({ errors: result.errors }, null, 2))` then `Deno.exit(1)`
- If success: `console.log(JSON.stringify(result.data, null, 2))`
- File not found: `console.error("Error: file not found: {path}")` then `Deno.exit(1)`

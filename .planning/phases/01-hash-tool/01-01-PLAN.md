Create `tools/hash-sections.ts` implementing the hash tool per D-01 through D-14. The file must work as both a library (imported by tests) and a CLI tool (via `import.meta.main` guard per D-11).

**Imports** (per D-07 -- inline npm specifiers with caret major pin):
```typescript
import { unified } from "npm:unified@^11.0.0";
import remarkParse from "npm:remark-parse@^11.0.0";
import { toString } from "npm:mdast-util-to-string@^4.0.0";
import type { Heading, Root } from "npm:@types/mdast@^4.0.0";
```

**Type definitions:**
```typescript
interface Section {
  heading: string;
  hash: string;
}

interface FileResult {
  path: string;
  sections: Section[];
}
```

**Exported function 1: `normalize(text: string): string`** (per HASH-03, Claude's discretion on exact implementation):
- Replace `\r\n` with `\n` (CRLF to LF)
- Replace `\r` with `\n` (CR to LF)
- Strip trailing whitespace per line: `replace(/[ \t]+$/gm, "")`
- Collapse 3+ consecutive newlines to 2: `replace(/\n{3,}/g, "\n\n")`
- Return the result

**Internal function: `sha256Hex8(text: string): Promise<string>`** (not exported -- internal helper):
- Encode text with `new TextEncoder().encode(text)`
- Hash with `crypto.subtle.digest("SHA-256", data)`
- Convert to hex string
- Truncate to first 8 characters
- Return the 8-char hex string

**Exported function 2: `hashSections(source: string): Promise<Section[]>`** (per D-01, HASH-01, HASH-02, HASH-04):
- Parse source with `unified().use(remarkParse).parse(source)` to get AST
- Iterate `tree.children` to collect indices where `node.type === "heading"` and `(node as Heading).depth === 2` (per D-13 -- only H2 boundaries)
- If no H2 indices found, return empty array (per D-04)
- For each H2 index:
  - Extract heading text via `toString(headingNode)` (per D-12 -- clean text, markup stripped)
  - Compute startOffset from `headingNode.position!.start.offset!` (per D-01 -- position offsets)
  - Compute endOffset from next H2's `position!.start.offset!`, or `source.length` for last section
  - Slice original source: `source.slice(startOffset, endOffset)` (per D-01 -- hash original bytes, NOT AST roundtrip)
  - Normalize the sliced text with `normalize()`
  - Hash with `sha256Hex8()` to get 8-char hex
  - Push `{ heading, hash }` to results
- Return results array

**Exported function 3: `hashFile(path: string): Promise<FileResult>`** (per HASH-06):
- Read file with `Deno.readTextFile(path)`
- Call `hashSections(source)` on the content
- Return `{ path, sections }` where `path` is the exact string passed in (per D-06 -- no normalization)

**CLI entrypoint** (guarded by `if (import.meta.main)` per D-11):
- If `Deno.args.length === 0`: print `Usage: hash-sections.ts <file1.md> [file2.md ...]` to stderr, exit code 1 (per D-05)
- Initialize `let hasError = false` and `const files: FileResult[] = []`
- Sequential for loop over `Deno.args` (per D-14 -- no parallelism):
  - Try: read file, hash sections, push to files array
  - Catch: set `hasError = true`, print error to stderr (per D-03). Check `Deno.errors.NotFound` for "file not found" message, otherwise generic error
- Print `JSON.stringify({ files }, null, 2)` to stdout (per HASH-05 schema)
- If `hasError`, call `Deno.exit(1)` (per D-03)

**CRITICAL constraints:**
- Do NOT import remark-stringify (eliminated by D-01)
- Do NOT normalize/resolve file paths in the JSON output (D-06)
- Do NOT use parallel processing (D-14)
- Do NOT create a deno.json file (D-07)

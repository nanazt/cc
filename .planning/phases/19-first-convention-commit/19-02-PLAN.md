Create `conventions/commit/hooks/validate.sh` -- a PreToolUse hook script that serves two purposes:

**1. Convention content injection (always, when convention file found):**
- Read the convention file and inject its full content via `additionalContext` in the JSON output
- Use `jq --rawfile` to safely read convention file content into a jq variable, NOT `$(cat)` with `--arg`. This prevents special characters (quotes, backslashes, newlines) in convention content from breaking JSON construction.
- The `additionalContext` value should be prefixed with: `COMMIT CONVENTION (follow these rules when writing commit messages):\n`
- This ensures Claude sees the convention rules before writing any commit message

**2. Mechanical validation (when commit message extractable from command):**
- Extract commit message from the command string (both `-m "..."` and `-m '...'` patterns)
- Validate these mechanically checkable rules and deny on failure:
  - **Conventional Commits format:** Subject must match `type(scope): description` or `type: description` (type without scope is valid)
  - **Allowed types:** feat, fix, docs, refactor, test, chore (deny other types)
  - **Scope pattern:** Scope must NOT be a bare number, `phase-N`, `phaseN`, or `state`
  - **Subject line length:** First line must be <= 100 characters
  - **GSD reference prohibition patterns:** Subject and body must NOT contain:
    - Phase references: `phase \d+`, `phase-\d+`
    - Plan IDs: patterns like `\d+-\d+-PLAN`, `plan \d+`, `plan \d+/\d+`
    - Requirement IDs: uppercase letters followed by dash and digits (e.g., `PIPE-01`, `MODEL-03`, `CASE-05`)
    - Decision IDs: `D-\d+`
    - Progress fractions: `\d+/\d+` when preceded by "plan" or "progress"
- On violation: output deny JSON with `permissionDecisionReason` explaining which specific rule was violated
- On no violation (or no message extractable): output allow JSON with `additionalContext`

**3. Command matching (gate):**
- Match both `git commit` and `gsd-tools.*commit` patterns in the command string
- Non-matching commands: exit 0 silently (no JSON output)

**4. Convention file discovery (dual-path per D-12 and Research Pattern 4):**
- Extract `cwd` from stdin JSON via `jq -r '.cwd'`
- Check `$CWD/conventions/commit/CONVENTION.md` first (publisher/source mode)
- Then check `$CWD/.claude/rules/cckit-commit.md` (consumer/installed mode)
- If neither exists: exit 0 silently (no injection, no validation -- per D-13 intent)

**5. Subsume validate-commit-scope.sh (Research Pitfall 5):**
- The new hook includes all checks from the old hook:
  - Bare number scope pattern: `^(phase-?)?[0-9]+$`
  - "state" scope blocking
- These are now part of the mechanical validation section above

**Implementation details:**
- Use `jq` for all JSON parsing (never eval stdin)
- Use `jq --rawfile` to read convention file content safely (handles quotes, backslashes, newlines without breaking JSON)
- Quote all shell variables to prevent injection
- Use `#!/bin/sh` (POSIX sh, not bash) for maximum portability
- NOTE: `jq --rawfile` requires jq 1.6+. System has jq 1.7.1 (verified in RESEARCH.md). If POSIX sh portability to systems without `--rawfile` is needed, alternative: `jq -Rs '.' < "$CONV_FILE"` to produce a JSON-safe string, then embed with `--argjson`.
- The script must be executable (chmod +x)
- Keep total script under ~100 lines to avoid complexity creep (per D-14 and Research Pitfall 2)
- For commit messages in heredoc format (`git commit -m "$(cat <<'EOF'...EOF)"`): the shell expands the heredoc before the hook sees it, so the `-m "..."` extraction should work. If extraction fails, fall back to injection-only (allow with additionalContext)

**Validation sequencing (per Research Pattern 3):**
- First extract and validate the commit message (if extractable)
- If invalid: output deny JSON and exit (additionalContext is NOT delivered on deny)
- If valid or not extractable: output allow JSON with additionalContext (convention injection)
- This means Claude always gets convention awareness on successful commits, and gets violation feedback on denied commits
